    Instructions for using GitHub

    Why should use Git/GitHub?

    Most of the time, when asking students about working on group assignments, where each member codes a specific part, how do they combine everything into a complete project?

    Most students usually respond: 'We'll meet at the library or some other place to piece it together or copy and share with each other.' There are also cases where they say, 'There's always someone carrying the team, so I just need to work on the documentation. :D

    This situation shows that most fresh graduates, who lack work experience, are often very confused about using Git. Some may not even know what Git is or how to work with it. From my perspective, this isn’t entirely their fault—it’s simply because they haven’t had the opportunity to use Git and therefore haven’t explored it. However, it will be their fault if they fail to learn about Git when working on a project that requires it.

    Recently, I had the opportunity to train a few interns and encountered similar situations as mentioned earlier. Therefore, I’d like to share some of my knowledge and experiences (from what I’ve learned and practiced) about Git in this article. I hope it will help others feel more confident working with Git and enhance their skills on the journey to becoming professional developers."

    What is Git?

    Git is a distributed version control system (Distributed Version Control System). Simply put, Git is a system that helps manage documents, source code, and more for a group of developers working on the same project. Git keeps a complete history of all changes made to the project's source code. Whether you edit a file, add a line of code, delete a line, or even leave an extra character somewhere—Git records all these actions. This makes it easier to investigate the root causes of system errors and to integrate code efficiently.

    Repository: Repository Simply put, it is a repository that stores all the necessary information to manage changes and history of entry project Repository of Git are divided into two types: remote repository and local repository.
